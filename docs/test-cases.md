# 财务报销测试样例

以下样例用于验证报销信息抽取结果。期望输出 JSON 遵循当前接口结构：

```json
{
  "报销类型": "string | null",
  "候选报销类型": [],
  "金额": "number | null",
  "日期": "string | null",
  "风险点": []
}
```

## 1. 差旅费

**输入文本**

```text
2026年4月18日我去北京出差，住宿和交通一共报销1280元。
```

**期望输出 JSON**

```json
{
  "报销类型": "差旅费",
  "候选报销类型": ["住宿费", "交通费"],
  "金额": 1280,
  "日期": "2026-04-18",
  "风险点": []
}
```

**为什么这样判断**

文本中明确出现“出差”，因此主报销类型判断为“差旅费”。“住宿”和“交通”作为差旅相关细分类，放入候选报销类型。金额和完整日期均可明确抽取，无明显风险。

## 2. 餐饮费

**输入文本**

```text
2026年3月12日部门客户接待用餐，餐费报销860元。
```

**期望输出 JSON**

```json
{
  "报销类型": "餐饮费",
  "候选报销类型": ["餐饮费"],
  "金额": 860,
  "日期": "2026-03-12",
  "风险点": []
}
```

**为什么这样判断**

文本中出现“用餐”和“餐费”，可明确归类为“餐饮费”。金额为 860，日期为 2026-03-12，信息完整且没有冲突。

## 3. 办公用品

**输入文本**

```text
2026年2月5日购买打印纸、文件夹和签字笔等办公用品，合计报销342.5元。
```

**期望输出 JSON**

```json
{
  "报销类型": "办公费",
  "候选报销类型": ["办公费"],
  "金额": 342.5,
  "日期": "2026-02-05",
  "风险点": []
}
```

**为什么这样判断**

“打印纸”“文件夹”“签字笔”“办公用品”都指向办公用品采购。当前接口类型中对应的是“办公费”，因此主类型和候选类型均为“办公费”。金额和日期明确。

## 4. 交通费

**输入文本**

```text
2026年4月30日拜访供应商往返打车，交通报销96元。
```

**期望输出 JSON**

```json
{
  "报销类型": "交通费",
  "候选报销类型": ["交通费"],
  "金额": 96,
  "日期": "2026-04-30",
  "风险点": []
}
```

**为什么这样判断**

“打车”和“交通”都明确指向交通类支出。文本只包含一个报销类型候选，金额和日期也完整，因此无风险点。

## 5. 日期缺失

**输入文本**

```text
酒店住宿报销600元，用于上周去深圳参加项目会议。
```

**期望输出 JSON**

```json
{
  "报销类型": "住宿费",
  "候选报销类型": ["住宿费"],
  "金额": 600,
  "日期": null,
  "风险点": ["日期无法从文本中明确判断"]
}
```

**为什么这样判断**

“酒店住宿”可明确判断为“住宿费”，金额为 600。但“上周”是相对时间，缺少可标准化为 YYYY-MM-DD 的具体日期，因此日期返回 null，并提示日期无法明确判断。

## 6. 金额缺失

**输入文本**

```text
2026年1月20日购买办公耗材，准备走报销流程。
```

**期望输出 JSON**

```json
{
  "报销类型": "办公费",
  "候选报销类型": ["办公费"],
  "金额": null,
  "日期": "2026-01-20",
  "风险点": ["金额无法从文本中明确判断"]
}
```

**为什么这样判断**

“办公耗材”可归类为“办公费”，日期明确。但文本没有出现具体报销金额或合计金额，因此金额返回 null，并提示金额缺失。

## 7. 发票金额和报销金额不一致

**输入文本**

```text
2026年5月1日上海出差交通报销1280元，但发票金额是1200元。
```

**期望输出 JSON**

```json
{
  "报销类型": "差旅费",
  "候选报销类型": ["交通费"],
  "金额": 1280,
  "日期": "2026-05-01",
  "风险点": ["报销金额与发票金额不一致", "文本中存在多个金额，需人工复核"]
}
```

**为什么这样判断**

“出差”说明主类型为“差旅费”，“交通”是候选细分类。报销金额为 1280，发票金额为 1200，两者不一致；同时文本中存在两个金额，需要人工复核。

## 8. 多个报销类型候选

**输入文本**

```text
2026年3月3日住宿和交通一共报销500元。
```

**期望输出 JSON**

```json
{
  "报销类型": null,
  "候选报销类型": ["住宿费", "交通费"],
  "金额": 500,
  "日期": "2026-03-03",
  "风险点": ["文本中存在多个报销类型候选，需人工复核", "报销类型无法从文本中明确判断"]
}
```

**为什么这样判断**

文本同时出现“住宿”和“交通”，但没有“差旅”或“出差”等明确大类。为避免强行归类，主报销类型返回 null，候选类型保留两个命中项，并提示需要人工复核。

## 9. 文本信息不足

**输入文本**

```text
麻烦帮我提交一下这笔费用。
```

**期望输出 JSON**

```json
{
  "报销类型": null,
  "候选报销类型": [],
  "金额": null,
  "日期": null,
  "风险点": ["报销类型无法从文本中明确判断", "金额无法从文本中明确判断", "日期无法从文本中明确判断"]
}
```

**为什么这样判断**

文本没有提供费用用途、金额和日期，无法判断任何核心字段。所有无法确认的字段返回 null，风险点说明信息不足。

## 10. 明显异常报销

**输入文本**

```text
2026年4月1日购买一支签字笔，办公用品报销50000元。
```

**期望输出 JSON**

```json
{
  "报销类型": "办公费",
  "候选报销类型": ["办公费"],
  "金额": 50000,
  "日期": "2026-04-01",
  "风险点": ["报销金额明显异常，需人工复核"]
}
```

**为什么这样判断**

“签字笔”和“办公用品”可判断为“办公费”，金额和日期明确。但单支签字笔报销 50000 元明显超出常识范围，应标记为异常报销并进入人工复核。
