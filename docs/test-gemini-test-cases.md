# Gemini 调用测试样例

本文档用于记录 `/extract-reimbursement` 背后的 `extractByLLM` 调用结果，重点验证 Gemini 接入链路。

## 本轮测试结论

- 测试日期：2026-05-06
- 调用入口：`src/llm.js` 的 `extractByLLM`
- Prompt 版本：Prompt v3
- 执行方式：`NODE_USE_ENV_PROXY=1 node -e ...`
- 模型配置：未额外设置 `GEMINI_MODEL`，使用代码默认值 `gemini-2.5-flash`
- 业务入口结果：10 条样例均返回了结构化 JSON，但都触发了 Gemini fallback
- 直接 SDK 诊断结果：`429 RESOURCE_EXHAUSTED`
- 底层原因：`gemini-2.5-flash` 免费层每日请求数达到 20 次上限

本轮没有拿到真实 Gemini 原生返回。以下“本轮接口输出”是当前代码在 Gemini 调用失败后回退到本地 mock 的输出。判断标准：如果 `风险点` 中出现 `Gemini 失效，已调用本地 mock 返回数据`，说明该条样例不是 Gemini 原生结果。

## 1. 差旅费

**输入**

```text
2026年4月18日我去北京出差，住宿和交通一共报销1280元。
```

**期望输出**

```json
{
  "报销类型": "差旅费",
  "候选报销类型": ["住宿费", "交通费"],
  "金额": 1280,
  "日期": "2026-04-18",
  "风险点": []
}
```

**本轮接口输出**

```json
{
  "报销类型": "差旅费",
  "候选报销类型": ["住宿费", "交通费"],
  "金额": 1280,
  "日期": "2026-04-18",
  "风险点": ["Gemini 失效，已调用本地 mock 返回数据"]
}
```

**判断原因**

核心字段与期望一致，但出现 fallback 风险点，本条不是 Gemini 原生结果。
